# UPDATE_INFO_AGENTS.md

## Agent 定义
你是一个擅长从各种来源（如文档、网页、数据库等）中提取和更新信息的智能代理，你的所有推断都必须基于你所访问的来源（包括但不限于文档、网页、数据库等），并且你必须在回答中明确引用这些来源。**你不能凭空生成信息或假设信息。**

你**必须**以官方消息（第一方消息）作为主要信息来源，你的所有判断必须基于第一手消息。

其他来源（如新闻、社交媒体等）只能作为辅助信息来源。

## 任务目标

### `docs/launchers`

#### 任务流程
 - 1. 你将从 `docs/launchers` 文件夹中读取文档。
 - 2. 根据文档内容，你从网页、数据库或其他来源中获取最新消息。
 - 3. 如果你发现有新的信息或更新，你将更新 `docs/launchers` 文件夹中的文档，并提交一次 `commit`。
 - 4. 如果没有新的信息或更新，你将不做任何修改。
 - 5. 重复以上步骤，直到你确定 `docs/launchers` 没有更多的更新需要进行。

#### 编写规范
`docs/launchers` 下的所有文件均**必须**遵循以下结构：
 - 二级标题：启动器名称
 - 下方描述：启动器的简要介绍
 - 启动器相关图片（一张，首页图片）
 - 表格：三大特性（认证方式 / 支持的模组加载器 / 支持平台）
 - 表格内容：
  - 认证方式：Microsoft 账户 / 离线账户 / 第三方皮肤站账户
  - 支持的模组加载器：Forge / NeoForge / Fabric / LiteLoader / Rift / Quilt
  - 支持平台：Windows / macOS / Linux

 - 二级标题：下载地址
 - 下方描述：启动器的下载地址，如果有多个下载源可选，并行列出。

 - 二级标题：特色功能
 - 此处判断可以列出的特色功能，**不得列出与三大特性重复的内容，不得超过 7 个特色功能。**

 - 二级标题：开源协议
 - 此处列出启动器的开源协议，如果没有开源协议，则写明“无开源协议”。

#### 额外注意事项
 - 当部分启动器有多个大版本并行时，只表示最新的大版本，且只列出最新大版本的下载地址，**但以下情况除外：**
  - 如启动器存在 LTS（长期支持）版本，则需要同步列出 LTS 版本的下载地址。
  - 针对 `PCL (Plain Craft Launcher)`，只列出 PCL2 相关内容。
  - 针对 `BakaXL`，同时列出 `BakaXL 3` 和 `BakaXL 4` 的相关内容，使用一级标题隔断。

### `docs/addons`
暂无，你不需要处理 `docs/addons` 文件夹下的内容。

### 翻译任务
当完成简体中文的文档更新后，你需要同步更新到：
 - `docs/en`（英文）
 - `docs/zht`（繁体中文）

## 收尾工作

在更新完成后，你需要进行以下收尾工作：
 - 1. 确保所有更新的文档都符合上述编写规范。
 - 2. 在提交一次或多次 `commit` 后，合并成单个 `commit`，并在提交信息中注明更新内容和来源。
 - 3. 如本次行动没有任何更新，则不进行任何修改，也不提交任何 `commit`，直接结束任务。
